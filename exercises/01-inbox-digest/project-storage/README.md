# Project storage (Exercise 1)

Your process writes here. You don't need to create anything — Cowork will.

- `updates/` — one short update file per open conversation (e.g., `c-maria.md`).
- `summary-YYYY-MM-DD.md` — the daily summary (also shown as a Cowork artifact).

To reset between runs, delete the files in `updates/` and any `summary-*.md`. Leave the
simulated inbox/CRM (in the sibling folders) untouched — those are read-only inputs.
